# Domain: XPU Kernel / Operator

## Path conventions

- torch-xpu-ops: `src/ATen/native/xpu/sycl/`, `src/ATen/native/xpu/`.
  Inside a PyTorch checkout: `third_party/torch-xpu-ops/<path>`.
- pytorch: `torch/`, `aten/src/ATen/`, `test/`, `c10/`, `torch/_dynamo/`,
  `torch/_inductor/`. Never `third_party/torch-xpu-ops/`.
- Never stage `third_party/xpu.txt`.

## CUDA cross-reference

CUDA counterparts live in `agent_space_xpu/pytorch/`:
- `src/ATen/native/xpu/<Op>.cpp` → `aten/src/ATen/native/<Op>.cpp`
- `src/ATen/native/xpu/sycl/<Op>Kernels.cpp` → `aten/src/ATen/native/cuda/<Op>.cu`

For ported tests, find the original under `agent_space_xpu/pytorch/test/` and
compare tolerances, dtypes, and device assumptions.

## Test path resolution

Working directory is `$PYTORCH_DIR/`.

| Input | Resolution |
|---|---|
| `test/xpu/*.py` | `third_party/torch-xpu-ops/test/xpu/*.py` |
| `third_party/torch-xpu-ops/...` | use as-is |
| CI dotted name | `third_party.torch-xpu-ops.test.xpu.test_ops_xpu` → `third_party/torch-xpu-ops/test/xpu/test_ops_xpu.py` |
| Upstream pytorch path | use as-is |

XPU test classes are often generated by `instantiate_device_type_tests`
(`<BaseClass>XPU`). Verify the class name before running.
