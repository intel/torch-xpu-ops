# Domain: XPU Kernel / Operator

Loaded by the orchestrator after `fix/triage` returns `"domain": "xpu-kernel"`.
Provides the XPU-specific knowledge that the generic pipeline skills defer to.

## Environment and build

Load `xpu-build-pytorch` for the full env/build workflow (oneAPI sourcing,
build command, arch flags, rebuild pitfalls).

## Nightly wheel

```bash
pip3 install --pre torch torchvision torchaudio \
  --index-url https://download.pytorch.org/whl/nightly/xpu
```

## Scratch directory

Use `agent_space_xpu/` at the repo root for temporary clones, scripts, and
build artifacts. It is git-ignored.

## Target repo path conventions

- **torch-xpu-ops** — fix lives under `src/ATen/native/xpu/sycl/`,
  `src/ATen/native/xpu/`, or any path relative to the torch-xpu-ops root.
  Inside a PyTorch checkout, these paths are under
  `third_party/torch-xpu-ops/`.
- **pytorch** — fix lives under `torch/`, `aten/src/ATen/`, `test/`, `c10/`,
  `torch/_dynamo/`, `torch/_inductor/`, or any top-level pytorch path (but NOT
  `third_party/torch-xpu-ops/`).

Never stage `third_party/xpu.txt`.

## Upstream cross-reference

When fixing XPU kernel code, compare against the CUDA implementation.
Reuse the checkout in `agent_space_xpu/pytorch/` if it already exists:

```bash
if [[ ! -d agent_space_xpu/pytorch/.git ]]; then
    git clone --depth 1 https://github.com/pytorch/pytorch.git agent_space_xpu/pytorch
fi
```

Use `agent_space_xpu/pytorch/` to read CUDA kernels, check upstream fixes, and verify
device-agnostic paths. Key mapping:

- `src/ATen/native/xpu/<Op>.cpp` → `aten/src/ATen/native/<Op>.cpp`
- `src/ATen/native/xpu/sycl/<Op>Kernels.cpp` → `aten/src/ATen/native/cuda/<Op>.cu`

## Failure categories

| Category | Description | Typical fix location |
|---|---|---|
| **XPU backend bug** | Bug in XPU kernel or operator code | `src/ATen/native/xpu/sycl/` |
| **Tolerance too tight** | Numerical precision mismatch vs CUDA | Adjust `atol`/`rtol` to match CUDA |
| **Edge case / numerical accuracy** | NaN/Inf, CPU-vs-XPU or fp32-vs-fp16 divergence, values near `finfo.max`/`min` | Compare against CUDA/CPU reference |
| **Skip decorator stale** | `@skipIfXpu`/`@expectedFailureXPU` but test now passes | Remove decorator — load `fix/pytorch-skip` |
| **Upstream regression** | Upstream code change broke XPU | `torch/`, `aten/`, or `third_party/torch-xpu-ops/` |
| **Test infrastructure** | Environment, import, or setup issue | Test file or CI config |

## Skip decorators

For all skip decorator operations (finding, removing, adding), load
`fix/pytorch-skip`.

## Test path resolution

Working directory is `$PYTORCH_DIR/`. Resolve test paths before running:

| Input format | Resolution |
|---|---|
| `test/xpu/*.py` | `third_party/torch-xpu-ops/test/xpu/*.py` |
| `third_party/torch-xpu-ops/...` | use as-is |
| CI metadata (dots) | `third_party.torch-xpu-ops.test.xpu.test_ops_xpu` → `third_party/torch-xpu-ops/test/xpu/test_ops_xpu.py`; class+method after last module component become the `-k` selector |
| Bare filename | `find $PYTORCH_DIR/ -name "<file>" -not -path "*/.git/*" -not -path "*/build/*" -not -path "*/.venv/*"`; prefer `third_party/torch-xpu-ops/` or `test/xpu/` on multiple matches |
| Upstream pytorch path (e.g. `test/nn/test_embedding.py`) | use as-is |

Many XPU test classes are generated by `instantiate_device_type_tests`
(`<BaseClass>XPU`, e.g. `TestSparseAnyXPU` from `TestSparseAny`). Validate:

```bash
grep "class TestSparseAny" <resolved_path>
grep "instantiate_device_type_tests" <resolved_path>
```
