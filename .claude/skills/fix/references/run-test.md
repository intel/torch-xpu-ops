# Run Test — Shared Reference

Used by `fix/reproduce` and `fix/verify`. Covers path resolution, running
tests, and interpreting results.

## Path resolution

Working directory is always `$PYTORCH_DIR/`. All test paths must be relative
to it.

| Input format | Resolution |
|---|---|
| `test/xpu/*.py` | `third_party/torch-xpu-ops/test/xpu/*.py` |
| `third_party/torch-xpu-ops/...` | use as-is |
| CI metadata (dots) | `third_party.torch-xpu-ops.test.xpu.test_ops_xpu` → `third_party/torch-xpu-ops/test/xpu/test_ops_xpu.py`; class+method after last module component become the selector |
| Bare filename | `find $PYTORCH_DIR/ -name "<file>" -not -path "*/.git/*" -not -path "*/build/*" -not -path "*/.venv/*"`; prefer `third_party/torch-xpu-ops/` or `test/xpu/` on multiple matches |
| Upstream pytorch path (e.g. `test/nn/test_embedding.py`) | use as-is |

Validate the file exists before running:
```bash
ls -la $PYTORCH_DIR/<resolved_path>
```
If not found → `CANNOT_VERIFY`.

## Dynamic test classes

Many XPU test classes are generated by `instantiate_device_type_tests`
(`<BaseClass>` + device suffix, e.g. `TestSparseAnyXPU` from `TestSparseAny`).
Validate with:
```bash
grep "class TestSparseAny" <resolved_path>
grep "instantiate_device_type_tests" <resolved_path>
```
If both exist, the dynamic class is valid. For static classes, grep the class
name directly.

For `-k` filters, validate with `--collect-only`:
```bash
pytest --collect-only -q "<file>" -k "<filter>" 2>&1 | tail -5
```
`0 items collected` → `CANNOT_VERIFY`.

## Running tests

Run from `$PYTORCH_DIR/`:
```bash
# pytest
pytest -xvs "<resolved_path>::ClassName::test_method"

# python unittest
python <resolved_path> ClassName.test_method
```
Keep the original format (pytest vs python). Timeout: 10 minutes.

## Result interpretation

### PASSED
- pytest exit 0, `N passed`, no `xfailed`, no `all skipped`
- unittest exit 0, `Ran N tests... OK`, actual tests ran

### FAILED
- pytest exit 1 with test failures
- unittest `FAILED` or `ERROR`
- Any actual test failure output
- All tests `xfailed` (expected failure = bug still present)

### CANNOT_VERIFY
- `collected 0 items` / `no tests ran`
- File not found on disk
- All tests skipped (cannot confirm either way)
- Timeout

## @skipIfXpu handling

If ALL tests are skipped by `@skipIfXpu` or similar:
- In `fix/reproduce`: temporarily remove the decorator, re-run, revert after.
  The skip is often the bug itself.
- In `fix/verify`: do NOT revert — the decorator should already be removed by
  `fix/implement` as part of the fix.
